# 技術詳細

## アーキテクチャ概要

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AIcheckers 技術スタック                           │
└─────────────────────────────────────────────────────────────────────┘

【1. 特徴抽出 (DINOv3)】

  入力画像 (224x224)
       │
       ▼
  ┌─────────────────────────────────────────────────────┐
  │  DINOv3 ViT-B/16                                    │
  │  (facebook/dinov3-vitb16-pretrain-lvd1689m)         │
  └─────────────────────────────────────────────────────┘
       │
       ▼
  201トークン出力:
  ┌───┬───┬───┬───┬───┬───────────────────────────┐
  │CLS│REG│REG│REG│REG│ PATCH1 ... PATCH196       │
  │ 0 │ 1 │ 2 │ 3 │ 4 │    5   ...    200         │
  └───┴───┴───┴───┴───┴───────────────────────────┘
    │                         │
    │ (768次元)               │ (196 × 768次元)
    ▼                         ▼

【2. 分類器入力 (774次元 = CLS + Patch Stats)】

  CLS Token (768d)        Patch Stats (6d)
  ┌──────────────┐        ┌─────────────────────────────────┐
  │ 画像全体の    │        │ 0: patch_mean    (AIスコア平均)  │
  │ グローバル特徴 │   +    │ 1: patch_max     (AIスコア最大)  │
  │              │        │ 2: patch_var     (AIスコア分散)  │
  └──────────────┘        │ 3: max - mean    (局所突出度)   │
                          │ 4: embed_var     (パッチ多様性)  │
                          │ 5: high_score%   (80%超の割合)   │
                          └─────────────────────────────────┘
                                    │
                          ┌─────────┴─────────┐
                          │ 768d分類器で      │
                          │ 各パッチを評価して │
                          │ 統計量を計算      │
                          └───────────────────┘
       │
       ▼
  ┌─────────────────────────────────────────────────────┐
  │  Linear Probe: nn.Linear(774, 2)                    │
  │  → [Real確率, AI確率]                               │
  └─────────────────────────────────────────────────────┘


【3. 学習時テクニック】

  ┌─────────────────────────────────────────────────────────────┐
  │ VAT (Virtual Adversarial Training)                          │
  │ ─────────────────────────────────────────────────────────── │
  │ • 全エポック適用                                             │
  │ • 「最も予測が不安定になる方向」にノイズを加えて学習          │
  │ • 決定境界を滑らかにする → 汎化性能向上                       │
  │                                                              │
  │   x_clean ──→ model ──→ p_clean                             │
  │      │                      │                                │
  │      + r_adv (敵対的ノイズ)   │ KL divergence を最小化        │
  │      │                      │                                │
  │   x_perturbed → model ──→ p_adv                             │
  └─────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────┐
  │ Consistency Regularization (epoch 5〜)                      │
  │ ─────────────────────────────────────────────────────────── │
  │ • embedding空間での疑似データ拡張                            │
  │ • 「弱いノイズ」と「強いノイズ」で同じ予測をさせる            │
  │                                                              │
  │   x + 弱ノイズ(σ=0.02) ──→ Teacher予測 (固定)               │
  │          ↓ 一致させる                                        │
  │   x + 強ノイズ(σ=0.08) + マスク ──→ Student予測             │
  └─────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────┐
  │ Entropy Minimization (epoch 20〜)                           │
  │ ─────────────────────────────────────────────────────────── │
  │ • 高信頼度(>90%)サンプルの予測をさらに尖らせる               │
  │ • 「曖昧な予測」を減らす                                     │
  │                                                              │
  │   予測確率: [0.92, 0.08] → もっと [0.98, 0.02] に近づける    │
  └─────────────────────────────────────────────────────────────┘


【4. 推論時テクニック】

  ┌─────────────────────────────────────────────────────────────┐
  │ TTA (Test-Time Augmentation)                                │
  │ ─────────────────────────────────────────────────────────── │
  │ • 元画像と左右反転画像の2回推論して平均                      │
  │                                                              │
  │   元画像     ──→ AI確率: 75%  ─┐                            │
  │                                 ├→ 平均: 76%                 │
  │   左右反転   ──→ AI確率: 77%  ─┘                            │
  └─────────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────────┐
  │ Temperature Scaling (T=1.5)                                 │
  │ ─────────────────────────────────────────────────────────── │
  │ • softmax前にlogitsをTで割る → 確率を滑らかに               │
  │ • 過信を防ぐ (99% → 95% など)                               │
  │                                                              │
  │   softmax(logits / 1.5)                                     │
  └─────────────────────────────────────────────────────────────┘


【5. Verdict判定 (5段階)】

  AI Score │ Verdict
  ─────────┼─────────────────
   80-100% │ AI DETECTED
   60-79%  │ HIGH ALERT
   40-59%  │ UNKNOWN
   20-39%  │ MINOR CAUTION
    0-19%  │ HUMAN CONFIRMED
```

## 要点まとめ

| 技術 | 目的 | 適用タイミング |
|------|------|----------------|
| CLS (768d) | 画像全体の特徴 | 特徴抽出 |
| Patch Stats (6d) | 局所的AI痕跡を集約 | 特徴抽出 |
| VAT | 決定境界を頑健に | 学習: 全エポック |
| Consistency | ノイズに強く | 学習: epoch 5〜 |
| EM | 曖昧な予測を減らす | 学習: epoch 20〜 |
| TTA | 反転も見て判断 | 推論時 |
| Temperature | 過信防止 | 推論時 |

## モデル仕様

- **ベース**: DINOv3 ViT-B/16 (facebook/dinov3-vitb16-pretrain-lvd1689m)
- **分類器**: Linear Probe (774 → 2)
- **精度**: ~97% (Val Acc)

## DINOv3トークン構造

```
[CLS, REG1-4, PATCH1-196] = 201トークン
  0     1-4      5-200
```
- パッチ抽出: `hidden_states[:, 5:5+196, :]`
- ⚠️ `hidden_states[:, 1:, :]` はREGトークンを含むので間違い

## 推論パラメータ

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| TTA_ENABLED | true | 水平反転で2回推論→平均 |
| TEMPERATURE | 1.5 | softmax平滑化 |
| HIGH_SCORE_THRESHOLD | 0.8 | パッチ統計計算用 |

## 環境

- GPU: GTX 1660 (6GB VRAM)
- DINOv3 VRAM: 0.34GB
- 推論時間: 70-120ms（TTA有効時）
